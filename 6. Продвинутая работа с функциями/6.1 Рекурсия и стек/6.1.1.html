Три варианта решения: 1. С использованием цикла. function sumTo(n) { let result = 0; for (let i = 1; i <= n; i++) { result += i; } return result; } 2. Через рекурсию, т.к. sumTo(n) = n + sumTo(n-1) for n > 1. function sumTo(n) { if (n === 1) { return n; } else { return n + sumTo(n-1); } } 3. С использованием формулы арифметической прогрессии. function sumTo(n) { return n * (n + 1) / 2; } Решение по формуле работает быстрее всех, т.к. оно использует только три операции для любого n, а цикл и рекурсия требуют как минимум n операций сложения. Вариант с циклом – второй по скорости. Он быстрее рекурсии, так как операций сложения столько же, но нет дополнительных вычислительных затрат на организацию вложенных вызовов. Поэтому рекурсия в данном случае работает медленнее всех. sumTo(100000); Uncaught RangeError: Maximum call stack size exceeded Некоторые движки поддерживают оптимизацию «хвостового вызова»: если рекурсивный вызов является самым последним в функции, без каких-либо других вычислений, то внешней функции не нужно будет возобновлять выполнение и не нужно запоминать контекст его выполнения. В итоге требования к памяти снижаются. Но если JavaScript-движок не поддерживает это (большинство не поддерживают), будет ошибка: максимальный размер стека превышен, так как обычно существует ограничение на максимальный размер стека.
